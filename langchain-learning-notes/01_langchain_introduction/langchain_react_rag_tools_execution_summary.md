# ReAct、RAG、Skills 与执行顺序（总览）

按 **固定顺序** 把 **ReAct、AI Agent、Tools/Skills、RAG、判断逻辑** 串成一条线；与 [白话总览](./langchain_plain_language_summary.md)、[架构与模块](./langchain_core_architecture_modules.md) 互补。

---

## 1. 核心概念一句话

- **ReAct = 推理（Reason）+ 行动（Act）**  
  是 AI Agent 自主解决问题时常见的一种 **工作模式**（先想再干、干了再想）。

- **AI Agent = 能按 ReAct 思路自主思考、选用工具、推进任务完成的智能体**  
  它不是某一个「工具」，而是 **总指挥 / 编排者**。

---

## 2. Skills / Tools 本质

- **本质**：**外部能力接口**（在本地或服务端执行，不在大模型权重里）。
- **作用**：让模型 **做到单靠生成文字做不到的事**。
- **例子**：查天气、调数据库、发消息、搜索、计算、读写文件等。

---

## 3. RAG 的作用、定位、本质

- **定位（概念上）**：可以把它理解成 **一类专门「查资料再交给模型」的能力**；在 Agent 语境下，常与 **Tool / 检索步骤** 一起出现（具体工程实现可能是「检索链 + LLM」或封装成一个 Tool，因框架而异）。
- **本质**：**检索外部知识 → 把检索结果放进上下文 → 由大模型生成回答**。
- **作用**：缓解大模型常见的三类问题：
  1. **知识过时**（训练数据有截止时间）
  2. **接触不到私有数据**（公司文档、笔记、代码库）
  3. **幻觉**（不确定时乱编）
- **特点**：RAG 侧 **只负责提供材料**；**最终回答仍由大模型生成**。

---

## 4. 完整执行顺序（Order，概念模型）

下面是一条 **便于记忆** 的 ReAct 式循环（真实框架里步骤名、次数会随实现变化）：

1. **用户提出问题**
2. **大模型第一次推理（Reason）**  
   判断：仅凭当前上下文，知识够不够？  
   - 够 → 直接回答  
   - 不够 → 决定调用工具（RAG 或其他 Skills）
3. **本地 / 服务端执行行动（Act）**  
   例如：走 RAG 检索、调 API、跑代码等
4. **把工具输出交回大模型**（进入新的上下文）
5. **大模型再次推理**  
   判断：资料够不够？  
   - 够 → 整理并回答  
   - 不够 → 再选工具或再检索，继续循环
6. **循环直到结束条件满足** → 输出最终答案

**要点**：每次工具调用后，通常都要 **回到大模型做下一步推理**，而不是工具单独对用户「结案」。

---

## 5. 什么时候会用到 RAG？（判断依据，概念上）

是否检索，往往由 **模型 + 提示 + Agent 策略** 共同决定；下面满足 **任一条** 时，**更适合**走检索增强（含 RAG）：

- 问题依赖 **私有 / 内部知识**（公司文档、个人笔记、项目资料）
- 需要 **较新的事实信息**（政策、新闻、数据）
- 需要 **高准确性、少胡编**
- 模型对答案 **明显不确定**

反之，很多场景 **不必**上 RAG，例如：

- 简单常识、简单计算  
- 纯逻辑推导（不依赖外部资料）  
- 开放性创作（不要求引用特定文档）

## 6. 举例：IDE 助手（如 Cursor）在思路上类似 ReAct Agent

你问：

> 帮我看一下我项目里的 Go 代码报错怎么修

一种典型的 **推理—行动** 节奏是：

1. **推理**：需要先看到项目里的代码与报错上下文。  
2. **行动**：调用 **读文件 / 搜符号 / 看诊断** 等工具（类似「针对代码库的检索与读取」）。  
3. **观察**：拿到代码与错误信息。  
4. **推理**：定位原因、构思修改。  
5. **行动**：给出补丁或编辑建议（可能再经多轮工具与推理）。  
6. **推理**：检查是否覆盖问题 → 输出最终结果。

可把这类产品理解为 **编程场景下的 ReAct 式 Agent**（具体实现远比这几步复杂）。

---

## 最精简背诵版（3 条）

1. **ReAct**：先推理 → 再调用工具 → 再推理 → … → 最终回答。  
2. **Skills**：外部操作接口；**RAG**：专门把外部资料送进上下文、再交给模型总结的一类能力（常和 Tool/链式检索一起出现）。  
3. **RAG 只供材料，不替代模型做最终表述；工具跑完后一般要回大模型继续推理。**
